
# Inline comments

Inline comments sit next to a section of code inside a function. They are generally discouraged—they often indicate that the described code should be extracted into its own function.

Use inline comments only to explain unintuitive parts of the code (e.g. non-obvious algorithms or the “why” behind a decision). They should not restate what the code is doing; the code should make that clear. For complex algorithms or business logic, prefer brief comments that explain the “why,” not the “what.”

**Checklist:** See the language-specific documentation rules in `documentation/checklists/` (Go, JavaScript/TypeScript, Python, Rust). Each has an Inline documentation or equivalent section with the full checklist.
